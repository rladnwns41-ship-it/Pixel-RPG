============================================================
  Medieval Realm — LAN(와이파이) 서버  사용법
  Firebase 안 씀 / 인터넷 안 씀 / npm 설치 안 함
============================================================

[필요한 거]
  - Node.js 만 깔려 있으면 됨 (https://nodejs.org)
  - 끝. 라이브러리 설치 1도 필요 없음.

[폴더에 있어야 하는 파일]
  s.js              ← 서버 (이거 실행)
  fb-shim.js        ← 가짜 firebase (게임이 LAN 서버로 말하게 해줌)
  game.js           ← 게임 로직 전체 (커스터마이징은 여기서!)
  admin_panel.html  ← 게임 화면(HTML). game.js / fb-shim.js 를 불러옴
  package.json

  ※ F12(개발자도구)·콘솔·우클릭·복사 전부 풀려 있음. 자유롭게 디버깅 가능.

------------------------------------------------------------
[1] 서버 켜기 (호스트 = 방 만드는 사람)
------------------------------------------------------------
  이 폴더에서 명령창 열고:

      node s.js

  그러면 바로 안 켜지고, 먼저 설정 메뉴가 뜸:

      ============================================
        Medieval Realm LAN Server - SETUP
      ============================================
         1)  Speed :  Normal  (100 ms)
         2)  Port  :  3000
         3)  Logs  :  detailed
      --------------------------------------------
        Type a number to change it, then Enter.
        Just press Enter to START the server.

   - 숫자 + Enter 로 옵션 변경:
       1 = 속도 바꾸기 (Smooth 50ms / Normal 100ms / Light 150ms 순환)
       2 = 포트 직접 입력
       3 = 로그 상세/간단 전환
   - 그냥 Enter 치면 서버 시작.
   - q + Enter 면 종료.

  서버가 켜지면 링크가 뜸:

      이 컴퓨터에서 테스트:  http://localhost:3000/
      📱 친구한테 줄 링크:   http://192.168.0.5:3000/   (← 너 IP)

  이 192.168... 링크를 같은 와이파이 친구한테 주면 됨.
  친구가 그 링크 열면 자동으로 너 서버에 접속 → 같이 플레이.

------------------------------------------------------------
[2] 링크마다 세계 분리 (원하면)
------------------------------------------------------------
  링크 뒤에 ?world=이름  붙이면 그 이름마다 완전 따로 세계임.
  데이터도, 보이는 플레이어도 안 섞임.

      http://192.168.0.5:3000/?world=myroom   ← A방
      http://192.168.0.5:3000/?world=friends  ← B방 (A랑 완전 분리)

  아무것도 안 붙이면 기본 세계(main) 로 들어감.
  → 친구들끼리 같은 링크(같은 world)면 같이 만남. 다르면 따로.

------------------------------------------------------------
[3] 저장은 어디에?
------------------------------------------------------------
  data/ 폴더에 월드마다 json 파일로 자동 저장됨.
      data/main.json
      data/myroom.json
  서버 꺼도 데이터 안 날아감. 다시 켜면 그대로 이어짐.

  ▶ 세계 초기화하고 싶으면: 그 월드 json 파일 지우고 서버 재시작.

------------------------------------------------------------
[4] 자주 묻는 거
------------------------------------------------------------
  Q. 친구가 접속 안 됨 / 계속 "서버 연결중" 만 뜸
   - 99%는 s.js 가 안 켜져 있어서임! 호스트가 'node s.js' 켰는지 확인.
   - F12 → Console 열어서 보면 상태가 글자로 뜸:
       [LAN] OK 서버 연결됨!   ← 이게 뜨면 정상 (Firebase 아님, s.js 임)
       [LAN] X 서버 연결 끊김   ← 이게 반복되면 s.js 가 안 켜졌거나 IP/포트 틀림
   - 같은 와이파이 맞는지, 방화벽이 3000 포트 막는지 확인
   - http:// 로 열어야 함 (https 아님, file:// 도 안 됨)

  Q. 옛날 'Cannot find module express' 에러가 남
   - 옛날 s.js 를 돌리고 있는 거임. 이 폴더의 새 s.js 로 덮어쓰기.
     새 s.js 는 http/crypto/fs/path/os(전부 Node 내장)만 씀 → 설치 0.

  Q. 포트 바꾸고 싶음
   - 윈도우:  set PORT=4000 && node s.js
   - 맥/리눅스: PORT=4000 node s.js

  Q. 진짜 Firebase 완전히 안 쓰는 거 맞음?
   - 맞음. 게임 안의 firebase 호출은 전부 fb-shim.js 가 가로채서
     너 컴퓨터의 s.js 로만 보냄. 구글/인터넷 0.

[5] 플레이어가 안 보일 때 (제일 중요)
============================================================
  증상: 블록(설치/파괴)은 같이 보이는데 플레이어 캐릭터만 안 보임.
  원인: 두 명이 "같은 닉네임" 또는 "같은 계정" 으로 접속함.
        게임은 '내 id 면 무시'(자기 자신 필터)가 있어서,
        닉이 같으면 서로를 자기 자신으로 보고 안 그림.
        (블록은 id 가 없어서 그냥 공유됨 → 그래서 블록만 보임)

  해결: 플레이어마다 "서로 다른 계정 + 다른 닉네임" 으로 가입해서 접속.
        - PC 와 폰에서 같은 계정으로 들어가면 안 보임 (같은 사람이니까)
        - 테스트하려면 계정 2개 만들어서 각각 다른 닉으로.
  (이제 가입 때 닉네임 중복은 자동으로 막힘)

[6] 서버 성능 커스텀 (호스트만)
============================================================
  s.js 맨 위 CONFIG 숫자를 바꾸면 접속한 모든 플레이어한테 적용됨:

    const CONFIG = {
      posRateMs: 100,   // 위치 전송 주기(ms). 낮을수록 부드럽고 트래픽↑
                        //   50=매우 부드러움, 100=기본, 150=가벼움
      idleMs:    2000,  // 가만히 있을 때 하트비트 주기(ms)
      timeoutMs: 6000,  // 이 시간(ms) 동안 조용하면 그 플레이어 사라짐
    };

  명령어로 임시 변경도 가능:
    윈도우:   set POS_RATE_MS=50 && node s.js
    맥/리눅스: POS_RATE_MS=50 node s.js

============================================================
[7] 서버(월드) 분리 확인
============================================================
  완전 분리됨. 분리 기준 = 링크의 ?world=이름.
    .../?world=a  와  .../?world=b  는 데이터/플레이어/맵 전부 따로.
    아무것도 안 붙이면 모두 'main' 으로 들어가서 같이 만남.
  data/<월드이름>.json 으로 월드마다 따로 저장됨.

============================================================
[8] 관리자 권한 (호스트)
============================================================
  서버를 켠 사람(호스트)이 자기 PC에서 http://localhost:포트/ 로
  접속하면 자동으로 관리자가 됨. (localhost = 호스트 본인이라 서버가 인정)

  게임 안에서 백슬래시 키( \ )를 누르면 관리자 패널이 열림.
   - 골드/아이템 지급 (본인=즉시, 다른 유저=우편함으로 전달)
   - 유저 목록 보기

  ※ 친구(다른 기기)한테도 관리자를 주고 싶으면:
     서버 켤 때 메뉴에서 4번 누르고 그 친구의 "게임 아이디"를 입력.
     (예: boss,master  처럼 쉼표로 여러 명 가능)
     또는 명령어로:  set ADMIN_IDS=boss,master && node s.js  (윈도우)

  주의: 호스트는 반드시 localhost 링크로 들어가야 관리자가 됨.
        자기 PC인데 192.168.x.x 링크로 들어가면 관리자 인식 안 될 수 있음.
